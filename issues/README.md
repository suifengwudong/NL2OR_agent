# Issues

## 说明

这个文件夹用于存放项目中遇到的 issues（问题）。这里的问题可以是 bug、功能需求、改进建议等。每个 issue 都应该包含以下信息：
- **标题**：简要描述问题的标题。
- **描述**：详细描述问题的内容，包括复现步骤、预期结果和实际结果。
- **标签**：为 issue 添加相关标签，如 bug、enhancement、question 等，以便分类和管理。
- **优先级**：根据问题的紧急程度和重要性，设置优先级，如高、中、低。

> 设计的缘由是方便远端协作和大模型辅助（因为它不能自己创建 Github issue），也可以方便后续总结和回顾。
